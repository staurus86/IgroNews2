# IgroNews — Полный набор функций

## Обзор системы

IgroNews — система мониторинга, анализа и переработки игровых новостей. Собирает новости из 15+ источников (RSS/HTML), анализирует через NLP, Keys.so, Google Trends и LLM, фильтрует дубликаты, оценивает виральность и позволяет переписывать статьи в разных стилях.

---

## 1. Дашборд

Главная страница со сводной информацией.

**Статистика:**
- Общее количество новостей в базе
- Количество по статусам: новые, на проверке, одобренные, отклонённые, обработанные
- Группировка дубликатов (одинаковые темы из разных источников)

**Таблица новостей:**
- Источник, заголовок (с переносом, без обрезки), дата публикации, дата парсинга
- Статус (new / in_review / approved / rejected / processed / duplicate)
- LLM-скор тренда
- Быстрые теги при наведении
- Фильтры: поиск по заголовку, по источнику, по статусу, лимит

**Массовые действия:**
- Выбрать все / снять все
- Отправить выбранные на проверку
- Одобрить / Отклонить выбранные
- Экспорт выбранных в Google Sheets (через очередь)

---

## 2. Проверка

Автоматическая оценка новостей по 6 критериям.

**Оценки (каждая 0-100):**
- **Качество** — длина текста, наличие заголовка, описания
- **Релевантность** — соответствие игровой тематике
- **Свежесть** — время с момента публикации
- **Виральность** — 18 триггеров (анонсы, скандалы, утечки, цифры, рекорды и др.) + Big Title + Calendar Events
- **Тональность** — positive / neutral / negative
- **Дедупликация** — выявление повторов из разных источников

**Виральные триггеры:** отображаются как цветные бейджи с весами (красный 40+, оранжевый 20+, синий <20).

**Фильтры:**
- По статусу, по тональности, по уровню виральности (High/Medium/Low/None)
- По минимальному общему скору
- Поиск по заголовку
- Сортировка по любому столбцу

**Действия:**
- Одобрить / Отклонить выбранные
- Авто-выбор прошедших проверку
- Открыть в Редакторе

---

## 3. Новости

Полная таблица всех новостей с данными анализа.

**Столбцы:**
- Источник, заголовок (полный, с переносом), дата публикации
- Статус (с цветным бейджем)
- Биграммы (TF-IDF ключевые слова)
- Keys.so — частота запросов + количество похожих ключей
- Google Trends — данные трендов
- LLM-рекомендация (публиковать / подождать / пропустить)
- LLM-скор тренда (0-100)
- Дата анализа
- Номер строки в Sheets

**Фильтры:** поиск, источник, статус, лимит (10-500), сортировка по столбцам.

**Массовые действия:**
- Одобрить / Отклонить выбранные
- Удалить выбранные
- **$ Анализировать выбранные** — батч-анализ через API (Keys.so + Trends + LLM)
- Экспорт выбранных в Sheets (через очередь)

**Действия для каждой новости:**
- $ Анализ — полный цикл: TF-IDF, Keys.so, Trends, LLM-прогноз
- Sheets — отправить в Google Sheets

---

## 4. Редактор

Рабочее пространство для переписывания новостей.

**Левая панель — список новостей:**
- Поиск, фильтр по источнику, фильтр по статусу
- Одиночный клик — выбрать для просмотра
- Чекбокс — отметить для слияния/батча
- Счётчик выбранных для слияния

**6 стилей переписывания:**
| Стиль | Описание |
|-------|----------|
| Новость | Информационная заметка — факты, без эмоций, инвертированная пирамида |
| SEO | SEO-оптимизация — ключи, подзаголовки, мета-описание, плотность 2-3% |
| Обзор | Авторский анализ — мнение, плюсы/минусы, прогнозы |
| Кликбейт | Максимальный CTR — интригующий заголовок, эмоциональные крючки |
| Кратко | Ультра-короткая заметка — 2-3 предложения, только суть |
| Соцсети | Пост для Telegram/VK/Twitter — эмодзи, хештеги, неформально |

**Язык:** русский или английский.

**Действия:**
- **$ Переписать** — переписать выбранную новость в выбранном стиле
- **$ Объединить** — слить несколько новостей в одну через LLM
- **Анализ** — полный анализ (виральность, свежесть, качество, релевантность, тональность, теги, тренды, Keys.so, биграммы)
- **$ Батч** — батч-переписать все отмеченные новости (через очередь)

**Панель анализа:** шкалы по критериям, виральные триггеры, настроение, теги, данные трендов и Keys.so.

**Результат (вкладка "Результат"):**
- Новый заголовок
- SEO Title (с длиной) + Meta Description (с длиной)
- Теги
- Переписанный текст рядом с оригиналом
- Кнопки: сохранить в Статьи, копировать текст, SEO, JSON, HTML

---

## 5. Статьи

Управление готовыми переписанными статьями.

**Список статей:**
- Заголовок, стиль, статус, дата создания
- Мульти-выбор чекбоксами
- Поиск, фильтр по статусу (draft / ready / published)

**Статусы:** draft -> ready -> published

**Редактор статьи:**
- Заголовок (с счётчиком символов)
- SEO Title (до 60 символов, с счётчиком)
- Meta Description (до 155 символов, с счётчиком)
- Теги
- Текст статьи
- Статус
- Оригинал (раскрывающийся блок)

**AI-действия ($):**
- Улучшить — общее улучшение текста
- Расширить — добавить деталей и глубины
- Сократить — убрать лишнее, оставить суть
- Грамматика — исправить ошибки
- SEO — добавить ключевые слова и структуру
- Вовлечение — сделать более увлекательным
- Переписать в стиле — выбрать любой из 6 стилей

**Экспорт:**
- Скачать DOCX — одна статья
- Массовый DOCX (ZIP) — выбранные статьи
- Удалить выбранные

## 6. Очередь

Мониторинг фоновых задач (переписка и Sheets экспорт).

**Типы задач:**
- Переписка (rewrite) — переписывание через LLM
- Sheets — экспорт в Google Sheets

**Статусы:**
- Ожидает (pending) — в очереди
- Обработка (processing) — выполняется
- Готово (done) — завершено успешно
- Ошибка (error) — ошибка при выполнении
- Отменено (cancelled) — отменено пользователем
- Пропущено (skipped) — дубликат в Sheets

**Функции:**
- Фильтр по типу и статусу
- Статистика по статусам (цветные бейджи с количеством)
- Отменить отдельную задачу
- Отменить выбранные
- Отменить все ожидающие
- Очистить завершённые / ошибочные / отменённые
- Автообновление каждые 15 секунд
- Бейдж на вкладке с количеством активных задач

---

## 7. Здоровье

Мониторинг работоспособности источников.

- Статус каждого источника (работает / ошибка)
- Количество статей за 24 часа
- Визуальная шкала активности
- Время последнего парсинга
- Минут с последнего обновления

---

## 8. Источники

Управление RSS/HTML источниками.

- Список всех источников с типом, URL, интервалом
- Добавить новый источник (имя, тип, URL, интервал, CSS-селектор)
- Редактировать / Удалить
- Перепарсить отдельный источник
- Перепарсить все источники

---

## 9. Промпты

Просмотр и редактирование LLM-промптов.

- Промпт прогноза тренда
- Промпт объединения новостей
- Промпт запросов Keys.so
- Сохранение изменений

---

## 10. Инструменты

Тестирование API-подключений.

- **Тест LLM** — отправить произвольный промпт и увидеть ответ ($)
- **Тест Keys.so** — проверить ключевое слово и увидеть частоту ($)
- **Тест Sheets** — проверить подключение к Google Sheets
- **Тест парсинга** — парсить любой URL и увидеть результат

---

## 11. Настройки

Конфигурация системы.

- LLM модель и Base URL (OpenRouter)
- API-ключи (LLM, Keys.so)
- Google Sheets ID
- Интервалы парсинга

---

## 12. Пользователи

Управление доступом.

- Список пользователей
- Добавить нового пользователя
- Изменить пароль
- Удалить пользователя
- Авторизация через подписанные куки (7 дней)

---

## Планировщик (фон)

Автоматические задачи (APScheduler):

- Парсинг источников по настроенным интервалам (15/30/60 мин)
- Обработка одобренных новостей каждые 10 минут (TF-IDF + Keys.so + Trends + LLM)
- Автозапись в Sheets **отключена** — экспорт только вручную через очередь

---

## API-конвенции

- Кнопки с пометкой **$** расходуют внешние API (LLM, Keys.so, Trends)
- Все массовые операции с Sheets и переписка идут через **очередь** с фоновой обработкой
- URL-дедупликация при экспорте в Sheets (проверка колонки B)
- Fallback на второй LLM-ключ при ошибке первого

---

## Технический стек

- Python: HTTPServer + APScheduler
- БД: SQLite (локально) / PostgreSQL (Railway)
- NLP: TF-IDF (scikit-learn), NLTK
- API: OpenRouter (LLM), Keys.so, Google Trends (pytrends)
- Экспорт: Google Sheets (gspread), DOCX (python-docx), ZIP
- Деплой: Railway (GitHub auto-deploy)
